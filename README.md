# World Cup CN Tracker

一个面向中文读者的世界杯赛事追踪网页，按北京时间展示赛程、比分、小组积分、出线趋势、淘汰赛路径和预测分析。

部署后，读者只需要打开 GitHub Pages 链接即可访问网页。

## 可以查看什么

- 今日赛程和按日期切换的比赛列表
- 全部赛程
- 已结束比赛比分
- 未开始比赛的北京时间开球时间
- 小组积分、出线形势和趋势
- 淘汰赛对阵路径
- Elo + 蒙特卡洛模型生成的预测分析
- 数据来源和最后更新时间

## 数据来源

核心赛程、比分、比赛状态和对阵信息来自 FIFA 官方接口：

```text
https://api.fifa.com/api/v3/calendar/matches?language=en&count=500&idSeason=285023
```

页面中的比赛时间统一转换为北京时间，并按北京时间自然日分组。第三方媒体链接只用于交叉核验或补充阅读，不作为主数据源。

## 数据准确性

项目以 FIFA 官方数据为准。同步脚本会拉取官方接口中的赛程、比分、状态和对阵信息，并写入 `data/matches.json`。

未结束比赛只显示北京时间开球时间，不提前改写比分；已结束比赛显示官方同步到的比分。如果官方接口尚未更新某场赛果，页面也会保持待更新状态。

## 预测分析说明

预测页使用两层逻辑：

- Elo 强度估计：根据已结束比赛更新球队相对强弱。
- 蒙特卡洛模拟：基于当前强度和未来赛程反复模拟，统计趋势概率。

预测内容由模型计算生成，仅供赛事观察和讨论参考，不构成确定性判断，也不应用于投注或其他高风险决策。

## 页面入口

- `index.html`：赛事看板首页
- `all-schedule.html`：全部赛程
- `prediction.html`：预测分析
- `elo-logic.html`：Elo 逻辑说明
- `monte-carlo-logic.html`：蒙特卡洛模拟说明

## 自动更新

仓库内置 GitHub Actions 工作流，会定时同步 FIFA 官方数据并发布到 GitHub Pages。

更新方式：

- 可在 GitHub Actions 中手动运行
- 默认每小时自动运行一次

GitHub Actions 的定时任务按 UTC 调度，实际执行时间可能存在延迟。

## 部署说明

将本项目作为独立 GitHub 仓库上传后，在仓库设置中启用 GitHub Pages，并将发布源设置为 `GitHub Actions`。

部署完成后，读者访问 GitHub Pages 链接即可使用网页，不需要下载项目，也不需要运行任何命令。

## 维护说明

本项目是静态网页项目，数据文件保存在 `data/` 目录，数据同步脚本保存在 `scripts/` 目录。常规维护主要包括：

- 检查 FIFA 官方接口是否可访问
- 检查 GitHub Actions 是否正常运行
- 检查页面中的最后更新时间
- 必要时调整球队中文名映射和预测模型说明
